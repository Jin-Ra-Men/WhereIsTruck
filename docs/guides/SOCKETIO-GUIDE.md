# Socket.io 초보자 가이드

**대상:** Socket.io(또는 실시간 통신)를 처음 접하는 분.  
**목표:** “이벤트 기반 실시간 통신”이 뭔지, “서버·클라이언트가 어떻게 주고받는지” 이해하는 것.

---

## 1. Socket.io란?

- **실시간 양방향 통신**을 위한 라이브러리입니다.
- 기본적으로 **WebSocket** 위에 올라가 있어, “연결만 맺어 두고, 서버↔클라이언트가 **이벤트 이름**과 **데이터**를 주고받는” 방식입니다.
- REST API는 “요청 한 번 → 응답 한 번”이지만, Socket.io는 “연결 유지 → 필요할 때마다 이벤트를 보냄”입니다.
- 우리 프로젝트에서는 **사장님 영업 시작/종료**, **위치 갱신**, **재고 공지** 등을 실시간으로 전달할 때 씁니다.

---

## 2. 핵심 개념 (처음 보는 분용)

### 2.1 연결 (Connection)

- 클라이언트가 **서버 주소**(예: `http://localhost:3000`)로 접속하면 **소켓**이 하나 생깁니다.
- 이 연결이 유지되는 동안 서로 **이벤트**를 보낼 수 있습니다.

### 2.2 이벤트 (Event)

- **이벤트 이름**(문자열) + **페이로드**(데이터) 로 주고받습니다.
- 예: 서버가 `truck:opened` 라는 이벤트에 `{ truck_id: 1, truck_name: "맛있는 떡볶이" }` 를 담아 보내면, 클라이언트는 그 이름으로 구독해 두었다가 받아서 처리합니다.
- “누가 누구에게”는 **방(Room)** 이나 **브로드캐스트**로 제어합니다. (전체에게 보내기, 특정 방만 보내기 등)

### 2.3 서버 vs 클라이언트

- **서버:** NestJS에서 **Gateway**로 Socket.io 서버를 띄웁니다. 클라이언트 연결을 받고, 이벤트를 수신·발송합니다.
- **클라이언트:** Flutter 앱 등에서 Socket.io 클라이언트 라이브러리로 같은 주소에 접속한 뒤, 이벤트를 **emit**(보내기) / **on**(받기) 합니다.

---

## 3. 이 프로젝트에서의 사용

- **이벤트 스펙:** **shared/api-spec/realtime-events.md** 에 이벤트 이름·페이로드가 정리되어 있습니다.
- **사장님 → 서버:** `truck:open`, `truck:close`, `truck:location`, `truck:stock_update` 등을 클라이언트가 **emit**.
- **서버 → 클라이언트:** `truck:opened`, `truck:location_updated` 등을 서버가 **emit** 해서 지도·목록을 갱신하거나 푸시와 연동합니다.
- NestJS에서는 `@WebSocketGateway()`, `@SubscribeMessage('이벤트명')`, `server.emit()` 등을 사용합니다.

---

## 4. REST와 비교 (감 잡기)

| 구분 | REST API | Socket.io |
|------|----------|-----------|
| 연결 | 요청마다 새로 연결 후 끊김 | 한 번 연결 유지 |
| 방향 | 클라이언트가 요청 → 서버가 응답 | 서버가 원할 때 이벤트 발송 가능 |
| 용도 | CRUD, 로그인, 조회 등 | 실시간 알림, 위치 갱신 등 |
